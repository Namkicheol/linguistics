# [임용영어학] Ch.2 Phrase Structure OX 40문항 — 해설 완전 정복

Phrase Structure 단원은 constituent를 "있다/없다" 수준에서 끝내지 않고, 그 constituent가 어떤 <span style="color:#3182ce">**rule**</span>과 <span style="color:#3182ce">**projection**</span>으로 조직되는지까지 묻는 파트입니다. 아래 40문항 해설은 <span style="color:#3182ce">**phrase structure rules**</span>, <span style="color:#3182ce">**X-bar theory**</span>, <span style="color:#3182ce">**complement vs adjunct**</span>, <span style="color:#3182ce">**DP hypothesis**</span>, <span style="color:#3182ce">**VP-internal subject hypothesis**</span> 순으로 읽으면 가장 효율적입니다.

---

## ① Phrase Structure Rules (Q1–Q8)

### Q1. Rewrite rule의 기본 정의 — ✅ O

<span style="color:#3182ce">**VP → V NP**</span> 같은 규칙은 VP를 그 immediate constituent로 전개하는 규칙입니다. 단순 어순표가 아니라 mother-daughter 관계를 명시하는 구조 규칙이라는 점이 핵심.

### Q2. S 아래 sister relation — ✅ O

<span style="color:#3182ce">**S → NP VP**</span>라면 NP와 VP는 S의 daughter이면서 서로 sister입니다. tree를 그릴 때 가장 먼저 잡아야 하는 골격.

### Q3. NP 안에 verb head? — ❌ X

<span style="color:#c53030">**⚠️ 틀린 진술입니다.**</span> NP 계열 구조의 head는 N 또는 D이며, verb가 nominal projection의 head가 될 수는 없습니다.

### Q4. Recursion — ✅ O

같은 계열의 phrase가 다른 phrase 안에 다시 들어가는 <span style="color:#3182ce">**recursion**</span> 덕분에 phrase structure는 무한한 길이의 구조를 생성할 수 있습니다.

### Q5. Rule은 hierarchy를 말하지 않는다? — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. phrase structure rule은 hierarchy를 전제로 하며, tree diagram은 그 hierarchy를 보이기 위한 표현입니다.

### Q6. Tree diagram의 필요성 — ✅ O

중간 constituent인 NP, VP, PP를 시각화하려면 tree diagram이 필요합니다. 문자열만으로는 intermediate node를 보여줄 수 없습니다.

### Q7. Branching node — ✅ O

두 개 이상 daughter를 갖는 node는 <span style="color:#3182ce">**branching node**</span>. c-command 정의에서도 계속 나오는 표현입니다.

### Q8. Terminal node — ✅ O

실제 단어가 놓이는 가장 아래 node가 terminal입니다.

## ② X-bar Schema (Q9–Q16)

### Q9. Head → X′ → XP — ✅ O

<span style="color:#3182ce">**X-bar theory**</span>의 최소 골격. head가 intermediate projection과 maximal projection을 차례로 만듭니다.

### Q10. Head가 범주 결정 — ✅ O

N이 head면 NP, V가 head면 VP. projection의 category label은 head에서 옵니다.

### Q11. Complement는 head의 sister — ✅ O

이 문항은 반드시 맞혀야 합니다. complement는 X′ 내부에서 head와 가장 가까운 관계를 맺습니다.

### Q12. Adjunct는 X′ 바깥 — ✅ O

adjunct는 head가 아니라 <span style="color:#3182ce">**X′**</span>에 붙습니다. 선택적 수식어라는 구조적 표현입니다.

### Q13. Specifier는 XP 층 — ✅ O

specifier는 X′와 결합하여 XP를 만듭니다. complement보다 바깥쪽.

### Q14. Head → adjunct → complement order? — ❌ X

<span style="color:#c53030">**⚠️ 반대입니다.**</span> 표준 구조는 head → complement → adjunct. complement가 head에 더 가깝습니다.

### Q15. XP는 maximal projection — ✅ O

X-bar 표기에서 XP가 가장 큰 projection 단계입니다.

### Q16. X-bar의 목적 — ✅ O

N, V, A, P, C를 공통 템플릿으로 설명하려는 generalization이 X-bar의 핵심 목적입니다.

---

## ③ Complement vs Adjunct (Q17–Q24)

### Q17. Selection 기준 — ✅ O

head가 요구하는 phrase라면 <span style="color:#3182ce">**complement**</span>일 가능성이 큽니다.

### Q18. Optionality — ✅ O

adjunct는 optional modifier이므로 제거해도 core grammaticality가 살아남는 경우가 많습니다.

### Q19. Recursion — ✅ O

adjunct는 여러 개 누적될 수 있지만, complement 자리는 head의 선택 정보에 의해 더 엄격하게 제한됩니다.

### Q20. Complement-before-adjunct order — ✅ O

> *the student <span style="color:#319795">**of linguistics**</span> <span style="color:#319795">**with a notebook**</span>*

이런 순서는 complement가 head에 더 가깝고 adjunct가 바깥이라는 분석과 잘 맞습니다.

### Q21. Coordination diagnostic — ✅ O

<span style="color:#c53030">**2022 A-7**</span> 유형과 연결됩니다. head가 요구하는 동일 범주와 자연스럽게 등위된다면 complement 분석을 지지할 수 있습니다.

### Q22. Noun은 PP를 선택하지 못한다? — ❌ X

<span style="color:#c53030">**⚠️ 자주 나오는 오개념**</span>. noun도 <span style="color:#3182ce">**PP complement**</span>를 가질 수 있습니다.

### Q23. Modifier left-behind pattern — ✅ O

대용이나 이동 시 modifier가 남아 있으면, 그 요소가 바깥쪽 adjunct일 가능성이 높습니다.

### Q24. 의미만의 구분? — ❌ X

complement와 adjunct 차이는 의미만이 아니라 attachment site, ordering, recursion에도 반영됩니다.

## ④ Specifier & DP Hypothesis (Q25–Q34)

### Q25. Specifier는 head-complement unit 바깥 — ✅ O

specifier는 XP의 외곽을 형성합니다.

### Q26. Subject as specifier — ✅ O

현대 phrase structure에서는 subject를 higher specifier로 분석하는 경우가 많습니다.

### Q27. Specifier = complement? — ❌ X

둘은 완전히 다른 structural position입니다.

### Q28. DP hypothesis — ✅ O

<span style="color:#3182ce">**DP hypothesis**</span>에서는 determiner가 D head, NP는 그 안쪽 complement/domain으로 분석됩니다.

### Q29. Wh-phrase와 Spec CP — ✅ O

Movement 단원과 직접 연결되는 포인트입니다. fronted wh-phrase의 landing site는 보통 <span style="color:#3182ce">**Spec CP**</span>.

### Q30. Edge position이면 모두 adjunct? — ❌ X

specifier도 edge에 있습니다. edge라는 이유만으로 adjunct로 볼 수 없습니다.

### Q31. DP as highest nominal projection — ✅ O

determiner-bearing nominal phrase를 DP로 본다는 게 핵심 주장.

### Q32. Determiner는 adjunct? — ❌ X

<span style="color:#c53030">**⚠️ DP hypothesis의 핵심을 반대로 뒤집은 선지**</span>. determiner는 head입니다.

### Q33. Overt D가 항상 있어야 하나? — ✅ O

null head 가능성을 열어 두기 때문에 overt determiner의 보편 실현을 요구하지 않습니다.

### Q34. CP layer — ✅ O

<span style="color:#3182ce">**CP**</span>는 complementizer가 head C로 들어가는 clause layer입니다. <em>that</em>, <em>whether</em> 같은 complementizer가 clause 외곽에 나타나면 CP 분석이 자연스럽습니다.

## ⑤ Clause Structure & Subject Origin (Q35–Q40)

### Q35. IP/TP — ✅ O

<span style="color:#3182ce">**IP/TP**</span>는 tense 또는 inflection 정보를 VP 위의 functional projection으로 표현합니다. lexical VP만으로는 tense, agreement, clause type을 충분히 표시하기 어렵기 때문에 필요한 층입니다.

### Q36. Small clause — ✅ O

> *consider [Mina intelligent]*

finite verb 없이도 predication 관계를 가지는 구성을 <span style="color:#3182ce">**small clause**</span>로 분석할 수 있습니다.

### Q37. Small clause는 structure로 표현 불가? — ❌ X

<span style="color:#c53030">**⚠️ 틀린 주장**</span>. finite tense가 없더라도 phrase structure는 predication unit을 충분히 표현할 수 있습니다.

### Q38. VP-internal subject hypothesis — ✅ O

subject가 처음부터 clause 바깥이 아니라 VP domain 내부에서 merge되고 이후 higher position으로 간다는 가설입니다.

### Q39. Lower origin이면 surface에서도 VP 안? — ❌ X

origin과 surface position은 구분해야 합니다. 이 가설의 핵심이 바로 그 분리입니다.

### Q40. Binary branching — ✅ O

현대 X-bar style analysis는 반복적인 two-way merge로 구현 가능합니다. binary branching과 잘 맞습니다.
