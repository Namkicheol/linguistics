# [임용영어학] Ch.7 Semantics OX 28문항 — 해설 완전 정복

<span style="color:#3182ce;">**Semantics**</span>는 결국 몇 개의 대립축으로 정리됩니다 — 의미를 보는 두 관점(<span style="color:#3182ce;">**sense**</span> vs <span style="color:#3182ce;">**reference**</span>, <span style="color:#3182ce;">**denotation**</span> vs <span style="color:#3182ce;">**connotation**</span>), 단어들 사이의 의미 관계(<span style="color:#3182ce;">**synonymy/antonymy**</span>, <span style="color:#3182ce;">**hyponymy/meronymy**</span>, <span style="color:#3182ce;">**homonymy/polysemy**</span>), 문장 사이의 진리 관계(<span style="color:#3182ce;">**entailment**</span> vs <span style="color:#3182ce;">**presupposition**</span>), 그리고 <span style="color:#3182ce;">**ambiguity**</span>·<span style="color:#3182ce;">**thematic roles**</span>·<span style="color:#3182ce;">**componential analysis**</span>까지입니다. 임용 의미론 OX는 거의 다 "이 둘 중 어느 쪽이냐"를 묻는 변별 문제입니다. 아래 28문항 해설로 한 번에 정리하세요.

---

## ① Sense vs Reference — denotation / connotation, deixis (Q1–Q4)

### Q1. *the morning star* / *the evening star* — 같은 reference, 다른 sense — ✅ O

The expressions *the morning star* and *the evening star* share the same <span style="color:#3182ce;">**reference**</span> (the planet Venus) yet differ in <span style="color:#3182ce;">**sense**</span>. 두 표현은 같은 referent(Venus)를 가리키지만 그것을 떠올리는 방식 — 아침에 보이는 별 / 저녁에 보이는 별 — 이 달라 <span style="color:#319795;">sense가 구별</span>됩니다. <span style="color:#3182ce;">**sense**</span>는 표현이 referent를 제시하는 방식(개념적 의미), <span style="color:#3182ce;">**reference**</span>는 실세계 대상과의 관계입니다. Frege의 고전적 구분입니다.

> **비교** — Sense: 내적·개념적 의미(제시 방식) / Reference: 실세계 referent와의 관계

### Q2. 같은 referent면 sense도 반드시 같다? — ❌ X

<span style="color:#c53030;">**⚠️ reference→sense 함정**</span>. If two expressions pick out the same referent, then they must also have <span style="color:#c53030;">the same sense</span> — 틀렸습니다. reference의 동일성이 sense의 동일성을 보장하지 않습니다. *Venus* / *the morning star*처럼 동일 referent를 서로 다른 방식으로 제시하면 <span style="color:#319795;">reference는 같아도 sense는 다릅니다</span>. 이것이 Frege가 지적한 핵심입니다.

### Q3. denotation = connotation? — ❌ X

<span style="color:#c53030;">**⚠️ 동의어 함정**</span>. *Denotation* and *connotation* are synonyms that pick out <span style="color:#c53030;">exactly the same aspect</span> of a word's meaning — 틀렸습니다. 둘은 구별됩니다. <span style="color:#319795;">**denotation**</span>은 단어가 적용되는 대상의 집합, 즉 문자적 중심 의미이고, <span style="color:#319795;">**connotation**</span>은 그 위에 얹히는 정서적·문화적 overtone입니다. 예를 들어 *thrifty*와 *stingy*는 denotation은 비슷하나 connotation이 정반대입니다.

> **비교** — Denotation: 문자적·중심 의미 / Connotation: 정서적·문화적 함축

### Q4. *here / now / this* — proximal deictic expressions — ✅ O

The expressions *here*, *now*, and *this* are <span style="color:#3182ce;">**deictic expressions**</span> that signal proximity (<span style="color:#3182ce;">**proximal**</span>) to the speaker. <span style="color:#3182ce;">**deictic expression**</span>은 발화 맥락에 의존해 지시 대상이 정해집니다. <span style="color:#319795;">*here, now, this*는 proximal(화자에 가까움)</span>을, *there, then, that*은 distal(화자에서 멈)을 나타냅니다.

## ② Synonymy & Antonymy (Q5–Q7)

### Q5. *alive / dead* — complementary antonyms — ✅ O

*alive* and *dead* are <span style="color:#3182ce;">**complementary antonyms**</span>: negating one entails the positive of the other, with no middle ground. <span style="color:#3182ce;">**complementary(binary) antonym**</span>은 한쪽을 부정하면 곧 다른쪽이 됩니다. <span style="color:#319795;">*not alive* = *dead*</span>이며 중간 단계가 없어 정도 부사·비교급이 부자연스럽습니다. *pass/fail*, *present/absent*도 같은 부류입니다.

> **비교** — Complementary: *alive/dead*, *pass/fail* — 중간 없음 / Gradable: *hot/cold*, *rich/poor* — 중간값 있음

### Q6. *employer / employee* — converse (relational) antonyms — ✅ O

*employer* and *employee* are <span style="color:#3182ce;">**converse (relational) antonyms**</span>: *A is B's employer* and *B is A's employee* paraphrase each other. <span style="color:#3182ce;">**converse antonym**</span>은 두 entity 사이의 관계를 정반대 시점에서 기술합니다. <span style="color:#319795;">*A is B's employer*가 참이면 *B is A's employee*도 반드시 참</span>이어서 서로 paraphrase입니다. *buy/sell*, *above/below*도 같습니다.

### Q7. *hot / cold* — complementary antonyms? — ❌ X

<span style="color:#c53030;">**⚠️ gradable 판별 함정**</span>. *hot* and *cold* are <span style="color:#c53030;">complementary antonyms, so there can be no intermediate value</span> — 틀렸습니다. *hot/cold*는 <span style="color:#319795;">**gradable antonym**</span>으로, 척도상의 대립이라 *warm, cool* 같은 중간값이 존재합니다. *hotter*, *very cold* 같은 비교급·정도 표현이 자연스럽습니다. 중간이 없는 complementary와 구별됩니다.

## ③ Hyponymy, Meronymy, Prototype (Q8–Q11)

### Q8. *sparrow* ⊨ *bird* — hyponym은 superordinate를 entail — ✅ O

*sparrow* is a <span style="color:#3182ce;">**hyponym**</span> of *bird*, and *I saw a sparrow* entails *I saw a bird*. <span style="color:#3182ce;">**hyponymy**</span>는 의미 포함(종류 관계)입니다. <span style="color:#319795;">hyponym(*sparrow*)이 superordinate(*bird*)를 entail</span>합니다 — 참새를 봤다면 새를 본 것입니다. 역방향(새를 봤다고 참새를 본 것은 아님)은 성립하지 않습니다.

### Q9. *I bought a flower* ⊨ *I bought a rose*? — ❌ X

<span style="color:#c53030;">**⚠️ 함의 방향 함정**</span>. Because *rose* is a hyponym of *flower*, *I bought a flower* entails <span style="color:#c53030;">*I bought a rose*</span> — 방향이 반대라 틀렸습니다. entailment는 hyponym→superordinate 한 방향입니다. <span style="color:#319795;">superordinate(*flower*)는 hyponym(*rose*)을 entail하지 않습니다</span> — 카네이션을 샀을 수도 있기 때문입니다.

### Q10. *wheel*은 *car*의 hyponym? — ❌ X

<span style="color:#c53030;">**⚠️ meronymy vs hyponymy 함정**</span>. A *wheel* is a hyponym of *car*, since a wheel is <span style="color:#c53030;">a kind of car</span> — 틀렸습니다. *wheel*은 *car*의 **부분**이지 **종류**가 아닙니다 → hyponymy가 아니라 <span style="color:#319795;">**meronymy**(부분–전체)</span> 관계입니다. *page∈book*, *petal∈flower*도 meronymy입니다.

> **비교** — Hyponymy: kind-of (*sparrow* ⊂ *bird*) / Meronymy: part-of (*wheel* ∈ *car*)

### Q11. 범주의 모든 멤버는 동등하게 전형적? — ❌ X

<span style="color:#c53030;">**⚠️ prototype 함정**</span>. In prototype theory, every member of a category is an <span style="color:#c53030;">equally typical</span> example — 틀렸습니다. <span style="color:#3182ce;">**prototype theory**</span>에서 범주 멤버십은 <span style="color:#319795;">**graded**</span>입니다. 모든 멤버가 동등한 게 아니라 더 중심적인 prototype과 주변적 멤버가 있습니다. *robin/sparrow*는 *bird*의 전형, *penguin/ostrich*는 주변적입니다.

## ④ Homonymy vs Polysemy (Q12–Q14)

### Q12. *bank* '강가' / '은행' — homonymy — ✅ O

*bank* 'edge of a river' and *bank* 'financial institution' have unrelated meanings, so they are a case of <span style="color:#3182ce;">**homonymy**</span> listed as separate dictionary entries. <span style="color:#3182ce;">**homonymy**</span>는 형태가 같고 의미가 무관한 경우로, <span style="color:#319795;">사전에 별개의 entry로 등재</span>됩니다. *bank*(강가) / *bank*(은행)는 역사적·의미적으로 무관하여 homonym입니다.

> **비교** — Homonymy: 무관한 의미 · 별도 entry / Polysemy: 관련된 의미 · 단일 entry

### Q13. *foot of the mountain* — polysemy — ✅ O

The *foot* in *the foot of the mountain* is related in meaning to the body-part *foot*, making this a case of <span style="color:#3182ce;">**polysemy**</span>. <span style="color:#3182ce;">**polysemy**</span>는 한 형태가 서로 관련된 여러 의미를 갖는 경우로, <span style="color:#319795;">사전에 단일 entry 아래 묶</span>입니다. *foot*(발) / *foot of the mountain*(산기슭)은 '아래·바닥'이라는 의미적 연결이 있어 polysemy입니다.

### Q14. *bright* '영리한' / '빛나는' — homonymy? — ❌ X

<span style="color:#c53030;">**⚠️ polysemy 혼동 함정**</span>. *bright* 'intelligent' and *bright* 'shining' have unrelated meanings, so the pair is a clear case of <span style="color:#c53030;">homonymy</span> — 틀렸습니다. *bright*의 '빛나는'과 '영리한'은 <span style="color:#319795;">은유적으로 연결된 관련 의미라 polysemy</span>입니다. 의미가 무관할 때만 homonymy입니다. 의미의 관련성 여부가 둘을 가르는 핵심 기준입니다.

## ⑤ Entailment vs Presupposition (Q15–Q19)

### Q15. entailment 정의 — p 참→q 참, q 거짓→p 거짓 — ✅ O

In an <span style="color:#3182ce;">**entailment**</span> relation, if p is true then q must be true, and if q is false then p must be false. <span style="color:#3182ce;">**entailment**</span>는 진리값으로 정의됩니다: <span style="color:#319795;">p가 참일 때 q가 반드시 참이고, q가 거짓일 때 p가 반드시 거짓</span>이면 p가 q를 entail합니다. 이 관계는 어휘·통사 구조가 보장합니다.

### Q16. p가 거짓이면 q도 거짓? — ❌ X

<span style="color:#c53030;">**⚠️ p-false 함정**</span>. If p entails q, then whenever p is false, q must also be <span style="color:#c53030;">false</span> — 틀렸습니다. entailment는 p 참→q 참, q 거짓→p 거짓만 보장합니다. <span style="color:#319795;">p가 거짓이면 q는 미정</span>입니다. 예를 들어 *I bought a dog*(거짓)이라도 *I bought an animal*은 참일 수 있습니다(고양이를 샀을 경우).

### Q17. *Liam stopped jogging* — presupposition은 부정해도 유지 — ✅ O

*Liam stopped jogging* and *Liam didn't stop jogging* both presuppose *Liam had been jogging* (<span style="color:#3182ce;">**constancy under negation**</span>). <span style="color:#3182ce;">**presupposition**</span>은 화자가 참으로 깔고 가는 배경 가정으로, <span style="color:#319795;">문장을 부정해도 살아남습니다</span>. *stop*은 change-of-state trigger라 긍정·부정 모두 *Liam had been jogging*을 전제합니다.

> **비교** — Entailment: 부정하면 사라짐(원문장에 딸림) / Presupposition: 부정해도 유지됨

### Q18. presupposition은 부정하면 사라진다? — ❌ X

<span style="color:#c53030;">**⚠️ negation test 함정**</span>. A presupposition <span style="color:#c53030;">disappears as soon as the sentence is negated</span> — 반대라 틀렸습니다. presupposition의 진단 기준이 바로 <span style="color:#319795;">**constancy under negation**</span>입니다. *My car is a wreck*와 *My car is not a wreck*는 둘 다 *I have a car*를 전제합니다. 부정 시 사라지는 것은 (원문장에만 딸린) entailment입니다.

### Q19. 능동문과 수동문은 서로 entail하지 않는다? — ❌ X

<span style="color:#c53030;">**⚠️ paraphrase 함정**</span>. An active sentence and its passive counterpart do <span style="color:#c53030;">not entail each other</span> — 틀렸습니다. paraphrase는 <span style="color:#319795;">**mutual entailment**</span>, 즉 서로를 entail하는 관계입니다. *The committee approved the plan*과 *The plan was approved by the committee*는 진리조건이 같아 서로를 entail합니다.

## ⑥ Ambiguity — lexical vs structural (Q20–Q21)

### Q20. *I saw the astronomer with the telescope* — structural ambiguity — ✅ O

*I saw the astronomer with the telescope* is <span style="color:#3182ce;">**structurally ambiguous**</span>, because the PP *with the telescope* can attach to either the NP or the VP. <span style="color:#3182ce;">**structural ambiguity**</span>는 동일한 단어열에 둘 이상의 구성 구조(트리)가 배정되어 생깁니다. <span style="color:#319795;">*with the telescope*가 NP(천문학자가 망원경 소지)에 붙거나 VP(망원경으로 봄)에 붙을 수 있어</span> 두 해석이 나옵니다.

### Q21. *He saw her duck* — structural ambiguity? — ❌ X

<span style="color:#c53030;">**⚠️ lexical 판별 함정**</span>. *He saw her duck* is ambiguous because the word *duck* has two senses, so this is <span style="color:#c53030;">structural ambiguity</span> — 틀렸습니다. 중의성이 단일 단어(*duck*: 명사 '오리' / 동사 '몸을 숙이다')의 여러 의미에서 비롯되므로 <span style="color:#319795;">**lexical ambiguity**</span>입니다. 구성 구조(트리)가 둘로 갈릴 때만 structural ambiguity입니다.

> **비교** — Lexical: 한 단어의 여러 sense / Structural: 한 단어열의 여러 트리

## ⑦ Thematic / semantic roles (Q22–Q25)

### Q22. *The thief broke the window with a brick* — Agent / Instrument — ✅ O

In *The thief broke the window with a brick*, *the thief* bears the <span style="color:#3182ce;">**Agent**</span> role and *a brick* bears the <span style="color:#3182ce;">**Instrument**</span> role. <span style="color:#3182ce;">**thematic role**</span>은 의미로 부여됩니다. 행위를 일으키는 <span style="color:#319795;">*the thief*는 Agent, 그 수단인 *a brick*은 Instrument</span>입니다. role은 grammatical function과 독립적입니다.

### Q23. *the vase*는 subject/object 상관없이 Theme — ✅ O

In both *The vase broke* and *Tom broke the vase*, *the vase* bears the <span style="color:#3182ce;">**Theme**</span> role even though it is subject in one and object in the other. <span style="color:#3182ce;">**thematic role**</span>은 grammatical function(subject/object)과 독립적으로 의미에 의해 부여됩니다. <span style="color:#319795;">*the vase*는 깨짐을 겪는 대상이므로 위치에 관계없이 Theme</span>입니다.

> **비교** — *The vase broke*: *the vase* = Theme (subject) / *Tom broke the vase*: *the vase* = Theme (object)

### Q24. *Sarah feared the storm* — *Sarah*는 Agent? — ❌ X

<span style="color:#c53030;">**⚠️ Experiencer 함정**</span>. In *Sarah feared the storm*, *Sarah* bears the <span style="color:#c53030;">Agent</span> role — 틀렸습니다. Agent는 행위를 능동적으로 일으키는 주체입니다. *fear*는 심리 상태를 나타내는 동사이고 <span style="color:#319795;">*Sarah*는 그 상태를 경험하는 Experiencer</span>입니다. 의도적 행위가 없으므로 Agent가 아닙니다.

### Q25. *Noah handed the key to the guard* — *the guard*는 Source? — ❌ X

<span style="color:#c53030;">**⚠️ Goal 함정**</span>. In *Noah handed the key to the guard*, *the guard* bears the <span style="color:#c53030;">Source</span> role — 틀렸습니다. <span style="color:#319795;">**Goal**</span>은 무언가가 이동해 가는 도착점, **Source**는 출발점입니다. *the key*가 *the guard*에게로 이동하므로 <span style="color:#319795;">*the guard*는 Goal</span>입니다. Source라면 *from*으로 표시되는 출발점이어야 합니다.

> **비교** — Goal: 도착점 (*to the guard*) / Source: 출발점 (*from Madrid*)

## ⑧ Componential analysis / Truth conditions (Q26–Q28)

### Q26. *man* vs *boy* — [±adult] 한 자질 차이 — ✅ O

In componential analysis, *man* and *boy* differ only in the value of the single feature <span style="color:#3182ce;">**[±adult]**</span>. <span style="color:#3182ce;">**componential analysis**</span>는 단어 의미를 이진 semantic feature로 분해합니다. <span style="color:#319795;">*man*=[+human,+adult,+male], *boy*=[+human,−adult,+male]</span>로, 둘은 [±adult] 한 자질에서만 다릅니다(둘 다 [+human, +male]).

### Q27. *A bachelor is unmarried* — analytic sentence — ✅ O

*A bachelor is unmarried* is an <span style="color:#3182ce;">**analytic sentence**</span> — true solely by virtue of the meanings of its words. <span style="color:#3182ce;">**analytic sentence**</span>는 세계를 확인하지 않고 단어 의미만으로 참이 보장되는 문장입니다. <span style="color:#319795;">*bachelor*의 의미 자체가 '결혼하지 않은'을 포함</span>하므로 필연적으로 참입니다. *A triangle has three sides*도 같습니다.

> **비교** — Analytic: 단어 의미만으로 필연적 참 / Synthetic: 세계 상태로 참·거짓 판정

### Q28. *The mayor owns three cars* — analytic sentence? — ❌ X

<span style="color:#c53030;">**⚠️ synthetic 함정**</span>. *The mayor owns three cars* is an <span style="color:#c53030;">analytic sentence</span>, true purely by virtue of word meanings — 틀렸습니다. <span style="color:#319795;">**synthetic sentence**</span>는 단어 의미만으로 진리값이 정해지지 않고 세계의 실제 상태에 따라 참·거짓이 결정됩니다. 시장이 차를 세 대 소유했는지는 <span style="color:#319795;">현실을 확인해야 알 수 있으므로 synthetic</span>입니다.

<span style="color:#dd6b20;">**현직쌤 팁**: Semantics OX는 거의 전부 "변별 함정"입니다. ① sense vs reference(같은 reference라도 sense는 다를 수 있음) ② entailment vs presupposition(부정 테스트 — entailment는 사라지고 presupposition은 살아남음) ③ hyponymy vs meronymy(kind-of vs part-of), homonymy vs polysemy(무관 vs 관련) ④ thematic role(문법 기능과 독립 — Agent/Experiencer, Goal/Source 구분) ⑤ analytic vs synthetic(단어 의미만으로 참 vs 세계 확인 필요)이 5대 빈출 축입니다. 단어쌍·문장쌍을 보면 항상 "이게 어느 쪽 범주인가, 그리고 함의 방향이 맞나?"를 먼저 물으세요.</span>
