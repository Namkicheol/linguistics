# Ch.4 Raising and Control — seem·try·believe·persuade 4분류 완전 정복 | 임용영어학 개념 정리

통사론에서 <span style="color:#3182ce;">**Raising**</span>과 <span style="color:#3182ce;">**Control**</span>은 표면 어순이 똑같은 *NP V (NP) to-VP*가 전혀 다른 두 구조에서 나온다는 점을 다룹니다. 핵심은 단 하나 — matrix 자리에 <span style="color:#3182ce;">**theta-role**</span>이 할당되는가입니다. 이 글은 <span style="color:#3182ce;">**SSR**</span>(seem) / <span style="color:#3182ce;">**SOR·ECM**</span>(believe) / <span style="color:#3182ce;">**SC**</span>(try) / <span style="color:#3182ce;">**OC**</span>(persuade) 4분류를 **① 개념 정의** → **② 기출 맥락** → **③ 키텀 비교** → **④ 수험 팁** 흐름으로 풀고, <span style="color:#3182ce;">**expletive**</span>·<span style="color:#3182ce;">**idiom chunk**</span>·<span style="color:#3182ce;">**voice transparency**</span> 진단으로 시험장에서 둘을 갈라내는 법을 정리합니다.

---

## ① 개념 정의

### Theta Criterion — 모든 판별의 토대

> **Each argument bears one and only one theta-role, and each theta-role is assigned to one and only one argument.** (<span style="color:#805ad5;">**Chomsky**</span> 1981)

<span style="color:#3182ce;">**Theta Criterion**</span>은 모든 argument(논항)가 정확히 하나의 <span style="color:#3182ce;">**theta-role**</span>(Agent, Theme, Goal, Experiencer 등)을 받고, 각 theta-role도 정확히 하나의 argument에만 할당된다는 원리입니다. theta-role은 <span style="color:#3182ce;">**D-structure**</span>에서 할당되고 movement 후에도 보존됩니다.

theta-role을 받는 자리는 <span style="color:#319795;">**theta-position**</span>, 받지 않는 자리는 <span style="color:#319795;">**non-theta position**</span>입니다. 바로 이 한 가지 구분이 Raising과 Control을 가르는 토대입니다.

- <span style="color:#3182ce;">**Raising verb**</span>(seem, appear …)의 matrix subject 자리는 <span style="color:#319795;">**non-theta position**</span> — 술어가 그 자리에 theta-role을 주지 않으므로, 의미가 빈 NP(<span style="color:#3182ce;">**expletive**</span>, <span style="color:#3182ce;">**idiom chunk**</span>)도 올 수 있다.
- <span style="color:#3182ce;">**Control verb**</span>(try, persuade …)의 matrix subject/object 자리는 <span style="color:#319795;">**theta-position**</span> — 술어가 theta-role을 주므로, 의미가 빈 NP는 못 온다.

결국 raising과 control의 모든 진단은 "matrix 자리에 theta-role이 할당되는가"로 환원됩니다.

---

### SSR — Subject-to-Subject Raising (seem)

> **Raising verbs** (*seem, appear, be likely, turn out, be unlikely, prove*) assign **no theta-role** to their surface subject. The subject of the embedded infinitival clause undergoes **NP-movement** to the matrix Spec,IP.

<span style="color:#3182ce;">**SSR**</span>에서 NP가 이동하는 이유는 <span style="color:#3182ce;">**Case**</span> 때문입니다. 하위절의 <span style="color:#3182ce;">**non-finite I**</span>(to)는 nominative Case를 주지 못하므로, 하위절 주어 NP는 Case를 받기 위해 matrix <span style="color:#319795;">**Spec,IP**</span>로 이동합니다. matrix subject 자리는 non-theta position이므로 NP는 이동 후에도 embedded predicate가 준 theta-role을 그대로 보유합니다.

구조 (이동 후): `[IP The student_i [I' Pres [VP seem [IP t_i to win ]]]]` — 하위절 주어가 matrix Spec,IP로 올라가고, 원위치에는 <span style="color:#3182ce;">**NP-trace**</span> t_i가 남습니다.

<span style="color:#3182ce;">**dummy-it**</span> paraphrase로 동의 관계가 확인됩니다:

- <span style="color:#319795;">*It seems that John is working.*</span> ≈ <span style="color:#319795;">*John seems to be working.*</span>

대표 정문 예:

- <span style="color:#319795;">*The students were found to have rashes.*</span>
- <span style="color:#319795;">*The supervisors turned out to be under investigation.*</span>
- <span style="color:#319795;">*The angry crowd is unlikely to leave the plaza.*</span>

`found` · `turned out` · `be unlikely` 모두 SSR — matrix subject에 theta-role이 없습니다.

---

### SOR·ECM — Subject-to-Object Raising / Exceptional Case Marking (believe)

> *believe, expect, want, report, consider, know* take an infinitival complement and exceptionally assign **objective (accusative) Case** to the subject NP of that complement — **Exceptional Case Marking (ECM)**.

<span style="color:#3182ce;">**believe**</span>류 동사는 infinitival 보문을 취하고, 하위절 주어 NP에 <span style="color:#3182ce;">**objective/accusative Case**</span>(목적격)를 예외적으로 부여합니다. 핵심은 matrix verb가 그 NP에 <span style="color:#c53030;">**theta-role을 주지 않는다**</span>는 것 — theta-role은 하위절 전체에 부여되므로 NP는 **하위절의 주어**로 행동합니다. 임용에서는 이를 <span style="color:#3182ce;">**SOR**</span>(Subject-to-Object Raising)으로도 부릅니다.

구조: `[IP Ed [VP believe [IP him[ACC] to be honest ]]]` — matrix V가 하위절 주어 *him*에 목적격을 across the clause boundary로 부여합니다.

표면상 *believe + NP + to-VP*는 object control(persuade)과 동일하지만, NP가 진짜 목적어가 아니라 하위절 주어라는 점에서 갈립니다. 이를 보이는 진단:

- <span style="color:#319795;">*Ed believes there to be a traitor.*</span> / <span style="color:#319795;">*Ed believes it to be raining.*</span> — expletive가 그 자리에 가능 → 하위절 주어
- <span style="color:#319795;">*Ed believes the coast to be clear.*</span> — idiom chunk 보존
- <span style="color:#319795;">*Ed believes the jury to have given the verdict.*</span> ≈ <span style="color:#319795;">*Ed believes the verdict to have been given by the jury.*</span> — voice transparency

`believe` vs `want` vs `expect` 미세 차이:

| 동사 | 특징 | 예 |
|---|---|---|
| **believe** | main clause passivize 가능 / 숨은 *for* 차단 | <span style="color:#319795;">*The story is believed to be false by Ed.*</span> &nbsp;|&nbsp; <span style="color:#c53030;">**\*I believe for the story to be false.**</span> |
| **want** | 숨은 *for*(Case) 때문에 main clause passivize 불가 (단 expletive·voice는 believe와 동일) | <span style="color:#c53030;">**\*Robert was wanted by Jean to go over there.**</span> |
| **expect** | subject control(앞)과 SOR(뒤) 둘 다 가능 | <span style="color:#319795;">*George expects to win.*</span> / <span style="color:#319795;">*George expects Sue to win.*</span> |

---

### SC — Subject Control (try)

> **Subject control verbs** (*try, plan, decide, hope, intend, threaten, prefer, refuse, be keen, be relieved*) **assign a theta-role to their subject**. The embedded infinitival clause has a **PRO** subject controlled by the matrix subject. **No movement.**

<span style="color:#3182ce;">**SC**</span>에서 matrix subject 자리는 <span style="color:#319795;">**theta-position**</span>이므로 expletive·idiom chunk가 올 수 없습니다. 하위절 주어는 이동의 흔적이 아니라 base-generated된 <span style="color:#3182ce;">**PRO**</span>이며, matrix subject와 <span style="color:#3182ce;">**coindex**</span>(control)됩니다.

구조: `[IP John_i [VP tried [CP PRO_i to resign ]]]` — matrix subject가 PRO를 control하고, 이동은 없습니다.

대표 정문 예:

- <span style="color:#319795;">*The researchers planned to start their project.*</span>
- <span style="color:#319795;">*The candidate was keen to support his opponent.*</span>
- <span style="color:#319795;">*I threatened to leave.*</span>

`planned` · `be keen` · `threaten` 모두 subject control — matrix subject에 theta-role을 부여합니다.

---

### OC — Object Control (persuade)

> **Object control verbs** (*persuade, order, force, allow, motivate, advise, remind, tell, convince*) assign a theta-role to their **matrix object**. The embedded PRO is controlled by the matrix **object** (not the subject).

<span style="color:#3182ce;">**OC**</span>에서 matrix object 자리는 <span style="color:#319795;">**theta-position**</span> — NP는 진짜 목적어이므로 expletive·idiom chunk가 올 수 없습니다. 하위절 PRO는 matrix object와 coindex됩니다.

구조: `[IP We [VP persuade Mary_i [CP PRO_i to resign ]]]` — matrix object *Mary*가 PRO를 control합니다.

이것이 <span style="color:#c53030;">**persuade(OC)**</span> vs <span style="color:#c53030;">**believe(SOR)**</span>를 가르는 핵심입니다. 둘 다 표면 *V + NP + to-VP*이지만 NP의 자격이 다릅니다:

- <span style="color:#c53030;">**\*Ed persuaded there to be a party.**</span> (expletive 불허 — 목적어가 theta-position)
- <span style="color:#319795;">*Ed believes there to be a problem.*</span> (expletive 허용 — 하위절 주어)
- <span style="color:#319795;">*Ed persuaded Brian to interview Melanie.*</span> ≠ <span style="color:#319795;">*Ed persuaded Melanie to be interviewed by Brian.*</span> — 두 문장 다 정문이지만 **의미가 다름**(≠). voice transparency 실패 — persuade가 목적어를 theta-mark하기 때문.
- <span style="color:#319795;">*John persuaded Sue that she should obey her parents.*</span> — persuade는 finite CP 보문도 취함 (SOR와 또 다른 차이).

---

### PRO — non-finite 절의 빈 주어

> **PRO** is the phonetically null subject of a non-finite (control) clause — an **empty category** that bears a theta-role from the embedded predicate. PRO is **base-generated** (not the result of movement).

<span style="color:#3182ce;">**PRO**</span>는 <span style="color:#3182ce;">**non-finite**</span>(infinitival to-)clause의 주어 자리에만 나타나며, 정형절 주어나 목적어 자리에는 못 옵니다. 예: <span style="color:#319795;">*I will try PRO to solve the problem.*</span> — PRO는 *solve*의 Agent argument로 theta-role을 받습니다.

| | Controlled PRO | Arbitrary PRO (PRO_arb) |
|---|---|---|
| 지시 | 특정 controller(matrix subject/object)와 coindex | 임의·일반적 지시 (≈ for anyone) |
| 예 | <span style="color:#319795;">*John_i tried [PRO_i to obey].*</span> | <span style="color:#319795;">*[PRO to go to college] is not essential.*</span> &nbsp;/&nbsp; <span style="color:#319795;">*It is not easy [PRO to love our neighbors].*</span> |

<span style="color:#3182ce;">**obligatory control**</span>은 controller가 반드시 있어야 하는 경우(try, persuade), <span style="color:#3182ce;">**optional control**</span>은 PRO가 controller 없이 arbitrary 해석을 가질 수 있는 경우입니다.

## ③ 키텀 비교

### 4분류 한눈에 — SSR / SOR·ECM / SC / OC

표면 어순 *NP V (NP) to-VP*가 같아도 아래 진단으로 갈립니다. 가장 빈출되는 판별 포인트입니다.

| 진단 | SSR (seem) | SOR·ECM (believe) | SC (try) | OC (persuade) |
|---|---|---|---|---|
| matrix subject에 theta-role | No | No | Yes | Yes |
| matrix object에 theta-role | — | No (목적격만) | — | Yes |
| 하위절 주어 | NP-trace (이동) | overt NP (목적격) | PRO | PRO |
| expletive there/it | OK | OK | <span style="color:#c53030;">\*</span> | <span style="color:#c53030;">\*</span> |
| idiom chunk 보존 | OK | OK | <span style="color:#c53030;">\*</span> | <span style="color:#c53030;">\*</span> |
| voice transparency | O | O | — | <span style="color:#c53030;">X</span> |
| movement | NP-movement | 없음 (Case in situ) | 없음 | 없음 |

### Raising vs Control 진단 3종

| 진단 | Raising (SSR·SOR) | Control (SC·OC) |
|---|---|---|
| **Expletive** there/it | <span style="color:#319795;">허용</span> — non-theta position | <span style="color:#c53030;">불허</span> — theta-position |
| **Idiom chunk** | <span style="color:#319795;">*The cat seems to be out of the bag.*</span> | <span style="color:#c53030;">**\*The cat tried to be out of the bag.**</span> |
| **Voice transparency** | 능동·수동 하위절이 동의 | 동의 깨짐 (특히 OC) |

### PRO vs NP-trace

| | PRO (control) | NP-trace (raising/passive) |
|---|---|---|
| 생성 | base-generated, 이동 아님 | NP-movement의 흔적 |
| theta-role | 하위절 predicate에서 보유 | moved NP가 보유 (trace 자체 X) |
| 자리 | non-finite 절 주어 (Caseless·PRO 허용) | Caseless 자리에서 이동 출발 |
| 동일지표 | controller가 control | moved NP와 coindex |

### ECM(believe) vs Object Control(persuade)

| | ECM / SOR (believe) | Object Control (persuade) |
|---|---|---|
| NP의 자격 | 하위절 주어 (목적격만 받음) | matrix 진짜 목적어 (theta-role 받음) |
| expletive | <span style="color:#319795;">OK</span> | <span style="color:#c53030;">\*</span> |
| voice transparency | <span style="color:#319795;">동의 O</span> | <span style="color:#c53030;">동의 X (의미 바뀜)</span> |
| finite CP 보문 | <span style="color:#c53030;">불가</span> | <span style="color:#319795;">가능</span> (*persuade Sue that …*) |

## ④ 수험 팁

### 답안 작성 패턴 (합격자 노트)

임용 Raising/Control 문항의 표준 답안 패턴입니다. 영어 그대로 외워 두면 서술형에서 바로 씁니다.

> **Raising**: *"The matrix predicate assigns no theta-role to its subject; the embedded subject NP undergoes NP-movement to matrix Spec,IP and retains its embedded theta-role."*

> **SOR·ECM**: *"The matrix verb assigns objective Case to the embedded subject across the clause boundary; the NP behaves as the subject of the infinitival clause (expletive / idiom / voice transparency)."*

> **Control**: *"The matrix verb assigns a theta-role to its subject (SC) / object (OC); the embedded infinitival clause has a PRO subject controlled by the matrix subject / object; no movement."*

### 시험장 판별 루틴

1. **Expletive 넣어 보기**: *there / it*가 들어가면 raising(SSR·SOR), 비문이면 control(SC·OC). 가장 빠른 1차 필터.
2. **Idiom chunk 넣어 보기**: *the cat … out of the bag*, *the coast … clear*가 관용 의미를 유지하면 raising.
3. **Voice transparency**: 하위절을 능동↔수동으로 바꿔 동의면 raising/ECM, 의미가 달라지면 object control(persuade).
4. **finite CP 보문 가능?**: *V + NP + that-clause*가 가능하면 control(persuade), 불가면 ECM(believe).

### 함정 정리

<span style="color:#c53030;">**⚠️ \*probable은 raising 불가, likely는 가능**</span>: <span style="color:#c53030;">**\*John is probable to pass the test.**</span> vs <span style="color:#319795;">*John is likely to pass the test.*</span> — 의미가 비슷해도 *probable*은 raising adjective가 아닙니다.

<span style="color:#c53030;">**⚠️ persuade(OC)와 believe(SOR)를 표면 어순으로 묶지 말 것**</span>: 둘 다 *V + NP + to-VP*이지만 NP가 persuade에선 진짜 목적어(theta-role O), believe에선 하위절 주어(목적격만). expletive·voice transparency로 반드시 검증.

<span style="color:#c53030;">**⚠️ want는 expletive·voice는 believe와 같지만 main clause passivize 불가**</span>: 숨은 *for*(Case) 때문. <span style="color:#c53030;">**\*Robert was wanted by Jean to go over there.**</span>

<span style="color:#c53030;">**⚠️ PRO를 NP-trace와 혼동 금지**</span>: control은 PRO(base-generated, 이동 없음), raising은 NP-trace(이동 흔적). PRO는 theta-role을 직접 보유하지만 NP-trace는 이동한 NP가 theta-role을 가집니다.

<span style="color:#c53030;">**⚠️ ECM은 theta-mark이 아니라 Case만 부여**</span>: believe가 하위절 주어에 주는 것은 objective Case일 뿐 theta-role이 아닙니다. theta-role은 하위절 predicate가 줍니다.

### 복습 포인트

- raising = matrix theta-role 無 → NP-movement(SSR) 또는 ECM Case(SOR). control = matrix theta-role 有 → PRO.
- 동사 빠른 분류: **seem·appear·be likely·turn out·be unlikely·prove**(SSR) / **believe·expect·want·report·consider·know**(SOR·ECM) / **try·plan·decide·hope·intend·threaten·prefer·refuse·be keen**(SC) / **persuade·order·force·allow·motivate·advise·remind·tell**(OC).
- 판별 3종(expletive·idiom chunk·voice transparency)은 전부 "matrix 자리에 theta-role이 있는가"를 묻는 같은 질문의 다른 형태임을 기억할 것.
