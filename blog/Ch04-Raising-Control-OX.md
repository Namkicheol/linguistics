# [임용영어학] Ch.4 Raising & Control OX 40문항 — 해설 완전 정복

<span style="color:#3182ce">**Raising**</span>와 <span style="color:#3182ce">**Control**</span>은 표면 어순이 같아도 구조가 전혀 다른 구문입니다. 모든 진단은 결국 "matrix 자리가 <span style="color:#3182ce">**theta-position**</span>인가"라는 한 기준으로 환원됩니다. <span style="color:#3182ce">**Theta Criterion**</span> · <span style="color:#3182ce">**SSR**</span> · <span style="color:#3182ce">**ECM**</span> · <span style="color:#3182ce">**PRO**</span>까지 아래 40문항 해설로 한 번에 정리하세요.

---

## ① Theta Criterion (Q1–Q4)

### Q1. Theta Criterion의 정의 — ✅ O

<span style="color:#3182ce">**Theta Criterion**</span>은 each argument bears one and only one theta-role, and each theta-role is assigned to one and only one argument를 요구합니다. argument와 theta-role 사이의 일대일 대응이 핵심입니다. 한 argument가 두 theta-role을 받거나, 한 theta-role이 두 argument에 할당되면 위반입니다.

### Q2. theta-role은 S-structure에서 할당된다? — ❌ X

<span style="color:#c53030">**⚠️ 핵심 함정**</span>. theta-role은 <span style="color:#3182ce">**D-structure**</span>에서 할당되며, movement 후에도 original position의 theta-role이 보존됩니다. 그래서 <span style="color:#3182ce">**raising**</span>으로 이동한 NP는 landing site(matrix Spec,IP, <span style="color:#3182ce">**non-theta position**</span>)가 아니라 original embedded position의 theta-role을 유지합니다. D-str = theta-role 할당, S-str = Case 점검입니다.

### Q3. raising verb의 surface subject 자리는 theta-position이다? — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. <em>seem</em> 같은 <span style="color:#3182ce">**raising verb**</span>의 surface subject 자리는 <span style="color:#3182ce">**non-theta position**</span>입니다 — 술어가 theta-role을 부여하지 않습니다. 그렇기 때문에 <span style="color:#319795">expletive · idiom chunk</span>가 그 자리에 올 수 있습니다. 만약 theta-position이라면 의미가 빈 NP는 허용되지 않을 것입니다.

### Q4. *The committee put the proposal* — argument 누락으로 비문 — ✅ O

<em>put</em>은 theme(NP)과 location(PP) 두 internal argument를 subcategorize합니다. <span style="color:#c53030">*The committee put the proposal*</span> is ungrammatical because the missing location PP leaves a theta-role unassigned, violating the <span style="color:#3182ce">**Theta Criterion**</span>. <span style="color:#c53030">**(2025 1차 B-5** 출제 각도)</span>

## ② Subject-to-Subject Raising / SSR (Q5–Q11)

### Q5. SSR의 정의: NP-movement + non-theta subject — ✅ O

<span style="color:#3182ce">**SSR**</span>: <em>seem</em>, <em>appear</em> 같은 <span style="color:#3182ce">**raising verb**</span>에서 embedded infinitival clause의 주어가 matrix Spec,IP로 <span style="color:#3182ce">**NP-movement**</span>하며, matrix verb는 그 subject에 theta-role을 부여하지 않습니다. raising verb는 one-place predicate로 하위절 전체만 취하고, 하위 비정형절 주어가 Case를 받기 위해 빈 matrix subject 자리로 이동합니다.

### Q6. *There seem to be several flaws* — expletive 진단 — ✅ O

expletive <span style="color:#319795">*there*</span>는 theta-role을 갖지 않습니다. <span style="color:#3182ce">**raising verb**</span>는 subject에 theta-role을 부여하지 않으므로 <em>there</em>를 허용합니다. <span style="color:#319795">*There seem to be several flaws in the design.*</span> 이것이 raising 판별의 핵심 진단 중 하나입니다.

### Q7. *The committee seems to review the budget* = control? — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. <em>seem</em>은 <span style="color:#3182ce">**raising verb**</span>입니다. <em>the committee</em>는 <span style="color:#3182ce">**PRO**</span>를 control하는 것이 아니라 <span style="color:#3182ce">**NP-movement**</span>로 embedded subject 자리에서 이동한 것입니다. control 구문이라면 matrix subject가 theta-role을 받아야 하는데, <em>seem</em>은 subject에 theta-role을 주지 않습니다.

### Q8. voice transparency → *seem*이 Agent theta-role 부여? — ❌ X

<span style="color:#c53030">**⚠️ 함정**</span>. 오히려 반대입니다. <em>The inspector seems to have examined the bridge</em>와 <em>The bridge seems to have been examined by the inspector</em>의 near-synonymy(<span style="color:#3182ce">**voice transparency**</span>)는 <em>seem</em>이 subject에 theta-role을 부여하지 **않는다**는 증거입니다. theta-role을 줬다면 두 문장의 주어가 달라 동의가 성립하지 않을 것입니다.

### Q9. *The candidate is probable to win* — *probable* 함정 — ✅ O

<em>likely</em>는 raising을 허용해 <span style="color:#319795">*The candidate is likely to win.*</span>이 정문이지만, <em>probable</em>은 raising predicate로 기능하지 않습니다. <span style="color:#c53030">*The candidate is probable to win the election.*</span> is ungrammatical. <em>probable</em>은 finite <em>It is probable that …</em> 형태로만 쓰입니다. 반복 출제된 함정입니다.

### Q10. *The cat seems to be out of the bag* — idiom chunk — ❌ X

<span style="color:#c53030">**⚠️ 함정**</span>. 비문이 아니라 정문입니다. <span style="color:#3182ce">**idiom chunk**</span>가 raising verb의 subject로 나타날 수 있다는 것이 raising의 증거입니다. raising verb는 subject의 의미 내용을 선택하지 않으므로 idiom chunk(<em>the cat … out of the bag</em>)가 그대로 보존됩니다. control verb였다면 idiom 의미가 깨져 비문이 됩니다.

### Q11. *It is believed the team to win* — Case 미수령 — ✅ O

<span style="color:#c53030">*It is believed the team to win the championship.*</span> is ungrammatical because <em>the team</em>이 Case를 못 받는 자리에 있어 비문입니다. raising/passive 술어는 외부논항이 없어 그 NP에 Case를 부여할 통로가 막혀 있으므로, <em>the team</em>은 정형 matrix clause의 nominative subject 자리(<span style="color:#319795">*The team is believed to win.*</span>)로 이동해야 합니다. <span style="color:#c53030">**(2024 1차 A-7** 출제 각도)</span>

## ③ Subject-to-Object Raising & ECM / SOR (Q12–Q19)

### Q12. ECM: 절 경계 넘어 objective Case 부여 — ✅ O

<span style="color:#3182ce">**ECM**</span>: <em>believe</em>, <em>expect</em>, <em>consider</em> 류 동사는 embedded infinitival clause의 주어에 <span style="color:#3182ce">**objective Case**</span>를 예외적으로 부여합니다. <em>I believe him to be honest</em>에서 <em>him</em>은 목적격을 받지만 <em>believe</em>의 의미상 목적어가 아니라 하위절(<em>to be honest</em>)의 주어입니다. matrix verb가 절 경계를 넘어 Case를 부여하는 것이 <span style="color:#3182ce">**Exceptional Case Marking**</span>입니다.

### Q13. *believes there to be an error* → *there*가 직접목적어? — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. <em>there</em>는 <em>believe</em>의 목적어가 아니라 하위절 주어입니다. <span style="color:#319795">expletive</span>는 theta-role을 못 받으므로 목적어일 수 없습니다. <em>The editor believes there to be an error in the draft</em>가 가능하다는 것은 그 자리가 theta-role 없는 자리(하위절 주어)임을 보여줍니다.

### Q14. *believe* + voice transparency 실패? — ❌ X

<span style="color:#c53030">**⚠️ 함정**</span>. 두 문장은 동의입니다(<span style="color:#3182ce">**voice transparency**</span>). <em>Ed believes the jury to have reached the verdict</em>와 <em>Ed believes the verdict to have been reached by the jury</em>는 진리조건적으로 동등합니다. <em>believe</em>는 NP가 아니라 하위절 전체에 theta-role을 부여하므로 하위절 내부 능동·수동 교체가 진리조건을 바꾸지 않습니다.

### Q15. *believes Tom that he has resigned* — single complement — ❌ X

<span style="color:#c53030">**⚠️ 함정**</span>. 비문입니다. <span style="color:#3182ce">**ECM**</span> 동사는 <span style="color:#3182ce">**single clausal complement**</span>만 취하며, 두 개의 internal argument(NP + that-clause)를 허용하지 않습니다. 정문: <span style="color:#319795">*John believes Bill to have slept.*</span> / 비문: <span style="color:#c53030">*The manager believes Tom that he has resigned.*</span>

### Q16. *Robert was wanted by Jean to leave* — *want* 수동화 차단 — ✅ O

<span style="color:#c53030">*Robert was wanted by Jean to leave early.*</span> is ungrammatical. <em>want</em>은 expletive·voice transparency에서는 <em>believe</em>와 같지만, 하위절 앞 보이지 않는 <em>for</em>가 Case를 처리하기 때문에 main-clause passivization(상위절 수동화)이 막힙니다. <em>believe</em>와의 결정적 차이입니다.

### Q17. *expect*은 SOR frame만 가진다? — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. <em>expect</em>은 subject control frame(<span style="color:#319795">*The coach expects to win.*</span>)과 SOR frame(<em>expects the team to win</em>) 둘 다 가능합니다. <em>The coach expects to win</em>은 <span style="color:#3182ce">**PRO**</span> 주어를 가진 control 구문으로 정문입니다.

### Q18. *likely* vs *believe* — Case 부여 대조 — ✅ O

<em>likely</em>는 형용사로 embedded subject에 Case를 줄 수 없어 raising(<span style="color:#319795">*Bill is likely to be late.*</span>)으로만 처리됩니다. <em>believe</em>는 <span style="color:#3182ce">**ECM**</span>으로 목적격을 부여할 수 있습니다. 대조: <span style="color:#c53030">*It is likely Bill to be late.*</span> vs <span style="color:#319795">*Bill is believed to be late.*</span> <span style="color:#c53030">**(2007 1차** 출제 각도)</span>

### Q19. *believe for the witness to be lying* — *for* 차단 — ✅ O

<span style="color:#c53030">*I believe for the witness to be lying.*</span> is ungrammatical because the complement of an <span style="color:#3182ce">**ECM**</span> verb is a bare clause without the complementizer <em>for</em>. ECM 구문의 보문은 complementizer 없는 절이어서 matrix verb가 직접 하위절 주어에 Case를 부여합니다. <em>for</em>가 들어가면 그 Case 부여 통로가 막혀 비문이 됩니다.

## ④ Subject Control (Q20–Q24)

### Q20. Subject Control: subject에 theta-role + PRO — ✅ O

<span style="color:#3182ce">**Subject control verb**</span>(<em>try</em>, <em>decide</em>)는 subject에 theta-role을 부여하고, embedded infinitival clause는 matrix subject가 control하는 <span style="color:#3182ce">**PRO**</span> subject를 가집니다. <em>John tried PRO to resign</em>에서 <em>try</em>는 <em>John</em>에 agent theta-role을 부여하고, 하위절 PRO는 <em>John</em>과 coindex됩니다. NP-movement는 일어나지 않습니다.

### Q21. *There tried to be a solution* — expletive 불허 — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. 비문입니다. <em>try</em>는 subject에 theta-role을 부여하는 <span style="color:#3182ce">**control verb**</span>이므로 expletive <em>there</em>가 올 수 없습니다(<span style="color:#c53030">*There tried to be a solution to the crisis.*</span>). control verb의 subject 자리는 theta-position입니다. 보문이 비정형이라는 점은 <em>seem</em>과 달리 raising/control을 가르지 못합니다.

### Q22. *The researcher decided to publish* = raising? — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. <em>decide</em>는 <span style="color:#3182ce">**subject control verb**</span>입니다. matrix subject <em>the researcher</em>에 agent theta-role을 부여하므로 raising이 아닙니다. <em>decide, try, plan, hope, intend</em>는 모두 subject에 agent theta-role을 부여하는 control verb이며, 하위절에는 <span style="color:#3182ce">**PRO**</span>가 있고 이동은 일어나지 않습니다.

### Q23. *The cat managed to be out of the bag* — idiom 불허 — ✅ O

idiom chunk는 고정 표현의 일부라 독립적 theta-role을 받을 수 없습니다. <span style="color:#3182ce">**control verb**</span> <em>manage</em>가 subject에 theta-role을 부여하므로 idiom 해석은 비문이 됩니다. <span style="color:#c53030">*The cat managed to be out of the bag.*</span> (idiom reading) raising verb였다면 정문입니다 — 이 대비가 raising/control 판별의 핵심입니다.

### Q24. *threaten* — subject control 술어 식별 — ✅ O

<em>The intern threatened to resign before the deadline</em>에서 <em>threaten</em>은 <span style="color:#3182ce">**subject control verb**</span>이므로 <em>resign</em>의 understood subject는 <em>the intern</em>이 control하는 <span style="color:#3182ce">**PRO**</span>입니다. <em>threaten, plan, prefer, be keen, be relieved</em>는 subject control 술어로, matrix subject가 하위절 PRO를 control하며 theta-role을 받습니다.

## ⑤ Object Control (Q25–Q30)

### Q25. Object Control: object에 theta-role + object가 PRO control — ✅ O

<span style="color:#3182ce">**Object control verb**</span>(<em>persuade</em>)는 matrix object에 theta-role을 부여하고, embedded <span style="color:#3182ce">**PRO**</span>는 subject가 아니라 그 <span style="color:#3182ce">**object**</span>가 control합니다. <em>We persuaded Mary PRO to resign</em>에서 <em>Mary</em>는 <em>persuade</em>의 진짜 목적어로 theta-role을 받고 하위절 PRO를 control합니다.

### Q26. *We persuaded there to be a meeting* — expletive 불허 — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. 비문입니다. <em>persuade</em>의 NP는 theta-role을 받는 진짜 목적어라 expletive <em>there</em>가 올 수 없습니다 — <em>believe</em>(ECM)와 정반대입니다. <span style="color:#319795">*We believe there to be a problem.*</span>은 가능하지만 <span style="color:#c53030">*We persuaded there to be a meeting on Friday.*</span>은 비문입니다. 두 동사는 표면 어순만 같습니다.

### Q27. *persuade* + voice transparency 성립? — ❌ X

<span style="color:#c53030">**⚠️ 함정**</span>. 두 persuade 문장은 동의가 **아닙니다**. <em>persuade</em>가 목적어를 theta-mark하므로 누구를 설득했는지가 의미를 바꿉니다. <em>We persuaded the doctor to examine Mary</em>(의사를 설득)와 <em>We persuaded Mary to be examined by the doctor</em>(Mary를 설득)는 다른 사건입니다 → <span style="color:#3182ce">**voice transparency**</span> 실패. ECM(<em>believe</em>)에서만 능동·수동 하위절이 동의입니다.

### Q28. *persuade Sue* vs *believe Sue* — 같은 구조? — ❌ X

<span style="color:#c53030">**⚠️ 핵심 구별**</span>. 두 구조는 다릅니다. <em>persuade</em>(object control)에서 <em>Sue</em>는 matrix object(theta-role 받음)이고, <em>believe</em>(ECM)에서 <em>Sue</em>는 하위절 주어(목적격만 받음)입니다. <span style="color:#319795">expletive · voice transparency</span>가 둘을 구별합니다.

### Q29. *persuade NP that S* — finite CP 보문 — ✅ O

<em>The officer persuaded the driver that he should pull over</em>의 정문성은 <em>persuade</em>가 finite CP complement도 취할 수 있음을 보여줍니다 — <span style="color:#3182ce">**ECM**</span> 동사와의 또 다른 차이입니다. object control verb <em>persuade</em>는 목적어 + 비정형 보문, 또는 목적어 + finite that-clause를 취할 수 있지만, ECM 동사는 single clausal complement만 취합니다.

### Q30. *advise* — object control 술어 식별 — ✅ O

<em>The principal advised the students to revise their essays</em>에서 <em>advise</em>는 <span style="color:#3182ce">**object control verb**</span>이므로 <em>revise</em>의 understood subject는 <em>the students</em>가 control합니다. <em>advise, order, force, remind, motivate, tell, allow</em>는 object control 술어로, matrix object가 하위절 PRO를 control하며 theta-role을 받습니다.

## ⑥ PRO (Q31–Q35)

### Q31. PRO의 정의: base-generated null subject + theta-role — ✅ O

<span style="color:#3182ce">**PRO**</span>는 non-finite control clause의 phonetically null subject로, embedded predicate에서 theta-role을 받고 movement가 아니라 base-generated됩니다. <em>I will try PRO to solve the problem</em>에서 PRO는 <em>solve</em>의 agent argument로 theta-role을 받습니다. PRO는 이동의 산물(trace)이 아니라 D-structure에서 생성된 empty category입니다.

### Q32. *The teacher hopes PRO will arrive* — PRO 분포 — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. <span style="color:#3182ce">**PRO**</span>는 비정형 to-clause의 주어 자리에만 나타납니다. 정형절 주어나 목적어 자리에는 올 수 없습니다. <span style="color:#c53030">*The teacher hopes PRO will arrive early.*</span>처럼 정형절 주어 자리에는 PRO가 올 수 없습니다.

### Q33. *[PRO to remain silent] is wise* — arbitrary PRO — ✅ O

matrix에 control할 NP가 없는 환경에서 <span style="color:#3182ce">**PRO**</span>는 임의·일반적 지시를 갖습니다(≈ for anyone). <em>[PRO to remain silent during the test] is sometimes the wisest choice</em>의 PRO는 specific controller 없이 arbitrary reference를 갖는 <span style="color:#3182ce">**arbitrary PRO**</span>(PRO_arb)입니다. <em>It is not easy PRO to love our neighbors</em>도 같은 경우입니다.

### Q34. NP-trace와 PRO는 같은 종류? — ❌ X

<span style="color:#c53030">**⚠️ 핵심 구별**</span>. 다릅니다. <span style="color:#3182ce">**PRO**</span>는 base-generated(이동 아님)이고 theta-role을 보유합니다. <span style="color:#3182ce">**NP-trace**</span>는 NP-movement가 남긴 흔적입니다. PRO는 control 절에 base-generated되어 theta-role을 받고, NP-trace는 raising/passive의 NP-movement가 Caseless 자리에 남긴 empty category로 이동한 NP와 coindex됩니다. 생성 방식이 근본적으로 다릅니다.

### Q35. *The volunteer hoped PRO to be selected* — controlled PRO — ✅ O

<span style="color:#3182ce">**subject control verb**</span> <em>hope</em>의 하위절 <span style="color:#3182ce">**PRO**</span>는 matrix subject <em>the volunteer</em>와 coindex되어 동일 지시를 갖습니다(controlled PRO). <em>selected</em>되는 대상은 자원자 자신입니다.

## ⑦ Diagnostics 종합 (Q36–Q40)

### Q36. SOR vs OC 판별: 같은 어순, 다른 진단 — ✅ O

<em>believe NP to-VP</em>와 <em>persuade NP to-VP</em>는 표면 어순이 같지만 <span style="color:#319795">expletive licensing</span>과 <span style="color:#319795">voice transparency</span>로 구별됩니다. <em>believe</em>(<span style="color:#3182ce">**SOR/ECM**</span>)는 둘 다 허용(NP가 하위절 주어)하고, <em>persuade</em>(<span style="color:#3182ce">**OC**</span>)는 둘 다 불허(NP가 theta-marked 목적어)합니다. 이것이 두 구문을 가르는 표준 진단입니다.

### Q37. expletive *there*가 *seem*·*try* 모두 허용? — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. <em>seem</em>(raising)은 expletive <em>there</em>를 허용하지만 <em>try</em>(control)는 불허합니다. <span style="color:#319795">*There seems to be a problem.*</span>은 정문이지만 <span style="color:#c53030">*There tried to be a problem.*</span>은 비문입니다. raising은 subject가 non-theta position이라 expletive를 허용하고, control은 theta-position이라 불허합니다. 보문이 비정형이라는 점만으로는 판별할 수 없습니다.

### Q38. *The suspect was said to have left* — passive *say* = raising — ✅ O

<em>say</em>의 수동형 <span style="color:#3182ce">**be said to**</span>는 하위절 주어를 matrix 주어로 raising하는 구문으로 쓰입니다. <span style="color:#319795">*The suspect was said to have left the country.*</span>는 <em>It was said that the suspect had left</em>과 동의이며, theta-role을 subject에 부여하지 않습니다. <span style="color:#c53030">**(2009 1차** 출제 각도)</span>

### Q39. idiom chunk가 네 술어 유형 모두에서 정문? — ❌ X

<span style="color:#c53030">**⚠️ 오답 포인트**</span>. <span style="color:#3182ce">**idiom chunk**</span> subject는 raising(<em>seem</em>)·ECM(<em>believe</em>)에서만 정문이고, control(<em>try</em>·<em>persuade</em>)에서는 비문입니다. raising·ECM 술어는 그 자리에 theta-role을 주지 않아 idiom chunk를 보존하지만, control 술어(<em>try</em>=subject, <em>persuade</em>=object)는 theta-role을 부여해 idiom 의미를 깨뜨립니다. theta-role 안 줌 → <span style="color:#319795">*seem, believe*</span> OK / theta-role 줌 → <span style="color:#c53030">*try, persuade*</span> 비문.

### Q40. raising vs control의 궁극적 기준 — ✅ O

<span style="color:#3182ce">**Raising**</span>과 <span style="color:#3182ce">**Control**</span>은 궁극적으로 matrix predicate가 relevant matrix position에 theta-role을 부여하는가로 구별됩니다. raising 술어는 matrix subject에 theta-role을 주지 않아(<span style="color:#3182ce">**non-theta position**</span>) NP-movement·expletive·idiom·voice transparency를 허용합니다. control 술어는 matrix subject/object에 theta-role을 주어(<span style="color:#3182ce">**theta-position**</span>) 이들을 막습니다. <span style="color:#dd6b20">**현직쌤 팁**: 모든 진단이 이 한 기준으로 환원된다 — 헷갈리면 "이 자리가 theta-role을 받나?"만 물어라.</span>
