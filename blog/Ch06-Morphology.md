# Ch.6 Morphology — 형태소·파생·굴절·합성어 완전 정리 | 임용영어학 개념 정리

형태론에서 <span style="color:#3182ce;">**Morphology**</span>는 단어가 어떤 최소 단위들로 이루어지고, 그 단위들이 어떤 규칙으로 결합해 새 단어를 만들거나 문법 기능을 표시하는지를 다룹니다. 핵심 축은 네 가지 — <span style="color:#3182ce;">**morpheme**</span>(최소 의미·기능 단위), <span style="color:#3182ce;">**inflection vs derivation**</span>(굴절 대 파생), <span style="color:#3182ce;">**word-formation processes**</span>(단어 형성 과정), 그리고 <span style="color:#3182ce;">**hierarchical word structure**</span>(계층적 단어 구조)입니다. 이 글은 <span style="color:#3182ce;">**free/bound morpheme**</span> 구분부터 <span style="color:#3182ce;">**allomorph**</span>, <span style="color:#3182ce;">**compounding**</span>의 <span style="color:#3182ce;">**endocentric/exocentric**</span> 대조, *unbuttonable* 같은 <span style="color:#3182ce;">**bracketing ambiguity**</span>까지를 **① 개념 정의** → **② 기출 맥락** → **③ 키텀 비교** → **④ 수험 팁** 흐름으로 정리합니다.

---

## ① 개념 정의

### Morpheme — free/bound, root/stem/base/affix

> **A morpheme is the minimal unit of meaning or grammatical function. A free morpheme can stand alone as a word; a bound morpheme cannot stand alone and must attach to another form — all affixes (prefixes and suffixes) in English are bound morphemes. A root is the core morpheme that carries the central meaning and cannot be further analyzed; a stem (base) is the form to which an affix attaches.**

<span style="color:#3182ce;">**Morpheme**</span>은 의미나 문법 기능을 가지는 **최소 단위**입니다. 혼자 단어로 설 수 있으면 <span style="color:#319795;">**free morpheme**</span>(*book*, *travel*), 반드시 다른 형태에 붙어야 하면 <span style="color:#319795;">**bound morpheme**</span>(*re-*, *-ist*, *-ed*, *-s*)입니다. 영어의 모든 <span style="color:#3182ce;">**affix**</span>(prefix·suffix)는 bound morpheme입니다. <span style="color:#3182ce;">**root**</span>는 더 쪼갤 수 없는 핵심 의미 형태이고, affix가 붙는 대상이 <span style="color:#3182ce;">**stem**</span>·<span style="color:#3182ce;">**base**</span>입니다. root는 가장 안쪽 핵심, stem/base는 affix가 붙는 시점의 형태라는 점을 구분합니다.

- <span style="color:#319795;">*unkindness*</span> = `un-`(prefix, bound) + `kind`(root, free) + `-ness`(suffix, bound) → 3 morphemes.
- <span style="color:#319795;">*players*</span> = `play`(free root) + `-er`(bound, agentive) + `-s`(bound, plural).
- <span style="color:#319795;">*cats*</span> = `cat`(free) + `-s`(bound). *cat* 단독은 free morpheme, *-s* 는 bound.
- <span style="color:#319795;">*book*</span> — 내부 구조 없는 단일 free morpheme.

free morpheme은 다시 <span style="color:#3182ce;">**lexical**</span>과 <span style="color:#3182ce;">**functional**</span>로 나뉩니다. lexical morpheme은 메시지의 '내용'을 나르는 noun·verb·adjective 등으로, 새로 잘 만들어지므로 <span style="color:#3182ce;">**open class**</span>입니다(*blog*, *email*, *to google*). functional morpheme은 conjunction·preposition·article·pronoun 같은 기능어로, 거의 추가되지 않으므로 <span style="color:#3182ce;">**closed class**</span>입니다(*the*, *of*, *because*, *them*).

### Inflection vs Derivation — 형태론 최대 대조

> **Inflection marks a grammatical function (number, tense, comparison, possession) without ever changing the grammatical category; English has exactly eight inflectional morphemes, all suffixes. Derivation makes a new word and may change the grammatical category, using either prefixes or suffixes.**

<span style="color:#3182ce;">**Inflection**</span>은 새 단어를 만들지 않고 단어의 문법 기능(수·시제·비교·소유)만 표시합니다. 품사를 **절대 바꾸지 않고**(category-preserving), 영어에서는 오직 8개이며 모두 suffix입니다. <span style="color:#3182ce;">**Derivation**</span>은 새로운 lexical word를 만들고 품사를 바꿀 수 있으며 prefix·suffix 모두 가능합니다. 핵심 판별: <span style="color:#319795;">**품사가 바뀌거나 새 단어가 생기면 derivation, 같은 단어의 문법형이면 inflection.**</span>

<span style="color:#3182ce;">**The 8 English inflectional morphemes**</span>(모두 suffix): noun `-'s`(possessive), `-s`(plural); verb `-s`(3rd-person singular), `-ing`(present participle), `-ed`(past tense), `-en`(past participle); adjective `-er`(comparative), `-est`(superlative).

| | <span style="color:#319795;">**Inflection**</span> | <span style="color:#319795;">**Derivation**</span> |
|---|---|---|
| 기능 | 문법 기능만 (수·시제·비교·소유) | 새 단어 생성, 품사 변경 가능 |
| 품사 변화 | **절대 없음** (category-preserving) | 흔함 (*good* A → *goodness* N) |
| 위치 | **suffix만** | prefix 또는 suffix |
| 목록 | **닫힘**: 정확히 **8개** | 열림 / 다수 |
| 생산성 | 매우 규칙적, 일반 적용 | 제한적·특이한 경우 많음 |

- Inflection: *teach → teaches / teaching / taught*; *tall → taller / tallest*; *dog → dog's / dogs*. 모두 품사 유지.
- Derivation: *teach*(V) → *teacher*(N); *happy*(A) → *happiness*(N); *kind*(A) → *unkind*(A, 품사 유지지만 새 단어).
- <span style="color:#dd6b20;">**함정 대조**</span>: <span style="color:#319795;">*taller*</span> 의 `-er`(comparative, **inflection**, adj→adj) vs <span style="color:#c53030;">*teacher*</span> 의 `-er`(agentive, **derivation**, V→N) — 동음이지만 다른 morpheme.

### Affix types — prefix/suffix, class-changing vs class-maintaining

> **By position, affixes are prefixes (before the base) or suffixes (after the base). By effect on word class, class-changing (category-changing) affixes alter the part of speech, while class-maintaining affixes keep it. Position and effect are independent axes.**

<span style="color:#3182ce;">**Affix**</span>는 위치로 prefix/suffix, 효과로 품사를 바꾸는지(<span style="color:#3182ce;">**class-changing**</span>)와 유지하는지(<span style="color:#3182ce;">**class-maintaining**</span>)로 나뉩니다. 영어 prefix는 대체로 class-maintaining(*un-*, *re-*, *dis-*)이고, 많은 suffix가 class-changing(*-ness*, *-ize*, *-ly*)입니다. 단, *un-*(adj→adj)처럼 prefix가 class-maintaining인 것과 *-able*(V→A)처럼 suffix가 class-changing인 것을 함께 보면 **위치와 효과가 별개 축**임이 분명합니다.

- Class-changing suffix: <span style="color:#319795;">*employ*(V) → *employment*(N)</span>; *beauty*(N) → *beautiful*(A); *quick*(A) → *quickly*(Adv).
- Class-maintaining prefix: <span style="color:#319795;">*legal*(A) → *illegal*(A)</span>; *write*(V) → *rewrite*(V).
- 위치 vs 효과 독립: *un-* 는 prefix이면서 class-maintaining; *-ize* 는 suffix이면서 class-changing.

> ⚠️ 이 축(POS 기준)은 음운 기반의 Class 1 / Class 2 affix 분류와는 **다른 개념**입니다. "class"라는 단어만 공유할 뿐입니다.

### Allomorphy — 한 morpheme의 조건 변이형

> **An allomorph is one of the variant phonetic forms of the same morpheme (same meaning/function). Allomorphs are typically phonologically conditioned — the surrounding sounds decide which form appears.**

<span style="color:#3182ce;">**Allomorph**</span>는 한 morpheme이 음운 환경에 따라 다른 형태로 실현된 것입니다. 의미·기능은 같고 **소리만 조건에 따라 갈립니다**(phonologically conditioned).

- <span style="color:#3182ce;">**Plural `-s`**</span>: `[s]` (voiceless 뒤, *books*) / `[z]` (voiced 뒤, *bags*) / `[əz]` (sibilant 뒤, *roses*) — 같은 plural morpheme의 3 allomorph.
- <span style="color:#3182ce;">**Past `-ed`**</span>: `[t]` (*kissed*) / `[d]` (*buzzed*) / `[ɪd]` (/t,d/ 뒤, *planted*).
- <span style="color:#3182ce;">**Negative prefix `in-`**</span>: place assimilation으로 4 allomorph — `in-` (elsewhere, *inactive*), `im-` (bilabial 앞, *impossible*, *imbalance*), `il-` (/l/ 앞, *illegible*), `ir-` (/r/ 앞, *irregular*).
- 부정관사 *a* (자음 앞) / *an* (모음 앞): *a* book / *an* apple.

allomorph들은 <span style="color:#c53030;">**의미가 다른 게 아니라**</span> 같은 의미·기능을 가지고 소리만 다릅니다. 이 점이 함정 포인트입니다.

### Word-formation processes — 단어 형성 과정

> **Derivation adds a derivational affix; compounding joins words; conversion (zero-derivation) shifts category with no affix; clipping shortens one word; blending fuses non-morphemic parts of two words; an acronym reads initials as a word (vs an initialism, read letter-by-letter); backformation removes a real or supposed affix.**

새 단어를 만드는 과정들입니다. 판별 핵심을 한 줄로 정리하면:

- <span style="color:#3182ce;">**Derivation**</span>: affix를 더함 — *kind → unkindness*.
- <span style="color:#3182ce;">**Compounding**</span>: 단어를 합침 — *love + seat → loveseat*.
- <span style="color:#3182ce;">**Conversion / zero-derivation / functional shift**</span>: affix 없이 품사만 바꿈 — *butter*(N) → *to butter*(V); 2음절어는 강세 이동 동반 *ímport*(N) / *impórt*(V).
- <span style="color:#3182ce;">**Clipping**</span>: 한 다음절 단어를 잘라냄 — *influenza → flu*, *advertisement → ad*, *facsimile → fax*.
- <span style="color:#3182ce;">**Blending**</span>: 두 단어의 **비형태소적 조각**을 섞음 — *smoke + fog → smog*, *information + entertainment → infotainment*.
- <span style="color:#3182ce;">**Acronym**</span>: 구의 머리글자를 **단어처럼 발음** — *NATO*, *laser*, *radar*. 글자를 따로 읽으면 <span style="color:#3182ce;">**initialism**</span>(*FBI*, *U.S.A.*).
- <span style="color:#3182ce;">**Backformation**</span>: 실재 또는 가정된 affix를 **떼어냄** — *editor → edit*, *donation → donate*, *babysitter → babysit*.
- <span style="color:#3182ce;">**Reduplication**</span>(영어에서는 minor): base 전체·일부 반복 — *so-so*, *bye-bye*.

<span style="color:#dd6b20;">**식별 핵심**</span>: clipping은 한 단어를 줄임, blending은 두 단어의 비형태소적 조각을 합침, acronym은 구의 머리글자를 단어처럼 읽음.

---

### Compounding — headedness, endocentric vs exocentric

> **A compound combines two or more words into a new word. In English the rightmost element is the head: it determines the compound's grammatical category and broad meaning. An endocentric compound is a hyponym of its head (a kind of the head); an exocentric compound denotes neither element.**

<span style="color:#3182ce;">**Compounding**</span>은 둘 이상의 단어를 합쳐 새 단어를 만듭니다. 영어에서 <span style="color:#3182ce;">**head는 오른쪽 요소**</span>이고, head가 전체의 품사·대략적 의미를 결정합니다(*black + bird* → *blackbird*, 일종의 **bird**).

- <span style="color:#3182ce;">**Endocentric compound**</span>: head의 하위어 — <span style="color:#319795;">*teapot*</span>(a kind of pot), *raincoat*(a kind of coat), *armchair*(a kind of chair).
- <span style="color:#3182ce;">**Exocentric compound**</span>(headless/bahuvrihi): 어느 쪽도 아님 — <span style="color:#319795;">*pickpocket*</span>(사람, pocket 아님), *scarecrow*(device, crow 아님), *highbrow*(사람).
- <span style="color:#3182ce;">**Category determination**</span>: 두 부분의 품사가 다르면 **마지막** 단어의 품사가 이김 — *handmade*(N+A → A), *pickpocket*(V+N → N), *overflow*(prep+V → V).
- <span style="color:#3182ce;">**Compound vs phrase**</span>: compound는 첫 요소에 강세(<span style="color:#319795;">*bláckboard*</span> 'chalkboard'), 대응 phrase는 둘째에 강세(<span style="color:#c53030;">*black bóard*</span> 'a board that is black').
- <span style="color:#3182ce;">**Inflection 위치**</span>: 복수·시제 표지는 첫 요소가 아니라 **복합어 전체**에 붙음 — <span style="color:#319795;">*toothbrushes*</span> (✓) / <span style="color:#c53030;">*teethbrush*</span> (✗).

### Hierarchical word structure — bracketing & ambiguity

> **Morphologically complex words are built in ordered steps, giving them hierarchical (layered) internal structure, shown by bracketing. Each affix's input/output category restricts the order, and a word can be ambiguous when its parts bracket two ways.**

<span style="color:#3182ce;">**Hierarchical word structure**</span>: 여러 affix를 가진 단어는 한꺼번에 만들어지는 게 아니라 **단계적**으로 형성되어 layered 구조를 가집니다. 각 affix의 input/output 품사 제약이 순서를 결정합니다. <span style="color:#3182ce;">*un-*</span> 는 서로 다른 두 morpheme입니다 — (a) `un-` + Adjective → Adjective ('not', *unhappy*); (b) `un-` + Verb → Verb ('reverse of', *untie*). 그래서 `un + V + able` 형태는 두 가지로 bracketing되어 <span style="color:#c53030;">**중의성**</span>이 생깁니다.

- <span style="color:#3182ce;">**Bracketing ambiguity**</span>: *unbuttonable*
  - <span style="color:#319795;">`[un-[button-able]]`</span> = 'not able to be buttoned' (adjectival *un-* + adjective *buttonable*).
  - <span style="color:#319795;">`[[un-button]-able]`</span> = 'able to be unbuttoned' (verbal *un-* + *-able*).
- <span style="color:#3182ce;">**Ordering fixed by selection**</span>: *unwearable* = `[un-[wear-able]]` 하나뿐. *-able*(V→A)이 먼저 동사 *wear* 에 도달해야 하고, adjectival *un-* 가 그 다음 형용사 *wearable* 에 붙음.
- *rewashable* = `[[re-wash]-able]`. *re-* 는 verb에만 붙으므로 *rewash* 를 먼저 만들고, 그 다음 *-able*; <span style="color:#c53030;">*\*re-[washable]*</span> 는 불가(*washable* 은 adjective).

여기에 더해, *-en*(causative, A→V) 같은 affix는 **monosyllabic + obstruent-final** base만 선택합니다(*whiten*, *soften* ✓ / <span style="color:#c53030;">*\*calmen*</span>, <span style="color:#c53030;">*\*curiousen*</span> ✗). 이런 selectional·phonological 제약이 derivation 순서와 가능성을 결정합니다.

## ② 기출 맥락

형태론은 임용에서 morpheme 분류·파생 대 굴절·단어 형성 과정·합성어 headedness를 묻는 영역으로 출제됩니다. 본 글은 **개념 위주**로 정리하며, 검증된 연도·문항 번호가 확인되지 않은 항목은 연도를 표기하지 않습니다(추정 금지).

토픽 색인 수준에서 형태론에서 출제 영역으로 확인되는 각도는 다음과 같습니다.

- <span style="color:#3182ce;">**Morpheme classification**</span> — free vs bound, lexical vs functional 구분.
- <span style="color:#3182ce;">**Derivational vs inflectional**</span> morpheme 구분 (특히 동음 affix 함정).
- <span style="color:#3182ce;">**Derivation / affixation**</span> 과 부정 prefix 선택 규칙(*in-/im-/il-/ir-*).
- <span style="color:#3182ce;">**Allomorph**</span> 와 phonological conditioning (plural *-s*, negative prefix).
- <span style="color:#3182ce;">**Conversion / zero-derivation**</span> (품사 전환, affix 없음).
- <span style="color:#3182ce;">**Compounding**</span> 과 word-formation process 순서·식별.
- <span style="color:#3182ce;">**Hierarchical structure / bracketing ambiguity**</span> (*un-…-able* 류).

> 연도가 검증되지 않은 항목은 연도를 표기하지 않았습니다. 위 각도는 출제 영역 확인용이며, 특정 연도·문항 번호·정답 wording은 박지 않습니다.

## ③ 키텀 비교

### Inflection vs Derivation — 한눈에

가장 빈출되는 판별 포인트입니다. 핵심은 "품사가 바뀌거나 새 단어가 생기는가"입니다.

| 항목 | Inflection | Derivation |
|---|---|---|
| 새 단어? | 아니오 (같은 단어의 형) | 예 |
| 품사 변경 | 절대 없음 | 가능 |
| 위치 | suffix만 | prefix·suffix 모두 |
| 개수 | 정확히 8개 (닫힘) | 다수 (열림) |
| 예 | <span style="color:#319795;">*taller*, *teaches*, *dogs*</span> | <span style="color:#319795;">*teacher*, *happiness*, *unkind*</span> |

### homonymy의 -er — 동음이지만 다른 morpheme

같은 형태 `-er` 가 두 개의 다른 morpheme입니다. 분류를 정확히 해야 합니다.

| 항목 | comparative *-er* | agentive *-er* |
|---|---|---|
| 유형 | <span style="color:#319795;">inflectional</span> | <span style="color:#319795;">derivational</span> |
| 품사 변화 | adj → adj (유지) | V → N (변경) |
| 예 | *tall → taller* | *teach → teacher* |

### Conversion vs Derivation — affix 유무

| 항목 | Conversion (zero-derivation) | Derivation |
|---|---|---|
| affix | <span style="color:#319795;">없음</span> | <span style="color:#c53030;">있음</span> |
| 품사 변경 | 있음 (N→V 등) | 있을 수도 |
| 예 | *butter*(N) → *to butter*(V) | *modern*(A) → *modernize*(V) |

⚠️ <span style="color:#c53030;">*modernize* 는 conversion이 아닙니다</span> — *-ize* suffix를 더했으므로 derivation입니다.

### Endocentric vs Exocentric — head의 하위어인가

| 항목 | Endocentric | Exocentric |
|---|---|---|
| head의 하위어? | 예 (a kind of head) | 아니오 |
| 의미 | head의 일종 | 어느 요소도 아님 (주로 사람·사물) |
| 예 | <span style="color:#319795;">*armchair* (a kind of chair)</span> | <span style="color:#319795;">*pickpocket* (a person)</span> |

### Class-changing vs Class-maintaining — 품사를 바꾸는가

| 항목 | Class-changing | Class-maintaining |
|---|---|---|
| 품사 변경 | 있음 | 없음 |
| 대표 | suffix 다수 (*-ness*, *-ize*, *-ly*) | prefix 다수 (*un-*, *re-*, *dis-*) |
| 예 | <span style="color:#319795;">*kind*(A) → *kindness*(N)</span> | <span style="color:#319795;">*kind*(A) → *unkind*(A)</span> |

> position(prefix/suffix)과 effect(class-changing/maintaining)는 별개 축입니다. *un-* 는 prefix이면서 class-maintaining, *-able* 는 suffix이면서 class-changing.

## ④ 수험 팁

### Word-formation process 식별법

과정을 묻는 문항은 "무엇을 했는가"로 즉시 분류합니다.

1. <span style="color:#dd6b20;">**affix를 더했는가?**</span> → derivation.
2. <span style="color:#dd6b20;">**두 단어를 통째로 합쳤는가?**</span> → compounding.
3. <span style="color:#dd6b20;">**affix 없이 품사만 바뀌었는가?**</span> → conversion.
4. <span style="color:#dd6b20;">**한 단어를 잘랐는가?**</span> → clipping (*flu* ← *influenza*).
5. <span style="color:#dd6b20;">**두 단어의 조각을 섞었는가?**</span> → blending (*smog* ← *smoke+fog*).
6. <span style="color:#dd6b20;">**머리글자인가?**</span> → 단어로 읽으면 acronym (*NATO*), 글자로 읽으면 initialism (*FBI*).
7. <span style="color:#dd6b20;">**affix를 떼어냈는가?**</span> → backformation (*edit* ← *editor*).

<span style="color:#c53030;">**⚠️ 단골 함정**</span>: *brunch* 는 clipping 아니라 **blending**(breakfast+lunch). *loveseat* 는 derivation 아니라 **compounding**(두 free word). *modernize* 는 conversion 아니라 **derivation**(*-ize* 추가).

### Bracketing ambiguity 분석 — *unbuttonable*

`un + X + able` 형태는 *un-* 가 두 morpheme이라 중의적일 수 있습니다.

> *unbuttonable* — `[un-[button-able]]` 'not able to be buttoned' vs `[[un-button]-able]` 'able to be unbuttoned'.

분석 순서는 <span style="color:#dd6b20;">**affix의 input/output 품사 제약**</span>으로 고정됩니다.

- *-able* 는 V→A: 반드시 동사에 붙습니다.
- adjectival *un-* 는 A→A, verbal *un-* 는 V→V.
- 따라서 *unwearable* 은 `[un-[wear-able]]` 하나뿐(*wear* 가 동사라 *-able* 먼저 부착 → adjectival *un-*).
- *rewashable* 은 `[[re-wash]-able]` 하나뿐(*re-* 는 동사 전용 → *rewash* 먼저, 그 다음 *-able*).

### Derivation 답안 패턴

서술형에서 구조·순서를 묻는 문항의 표준 답안 방향입니다. 영어 그대로 외워 두면 바로 씁니다.

> *"The word X is structurally ambiguous: [un-[V-able]] means 'not able to be V-ed', while [[un-V]-able] means 'able to be un-V-ed'. The two readings arise because un- is two distinct morphemes (adjectival 'not' / verbal 'reverse')."*

> *"The order is fixed by selectional restriction: -able attaches only to a verb (V→A), and adjectival un- attaches only to an adjective (A→A); therefore the only well-formed bracketing is [un-[V-able]]."*

### Allomorph 조건 정리

allomorph 문항은 "어떤 소리 뒤인가"로 형태를 고릅니다.

| morpheme | 조건 | 형태 | 예 |
|---|---|---|---|
| plural *-s* | voiceless 뒤 | [s] | *books* |
| | voiced 뒤 | [z] | *bags* |
| | sibilant 뒤 | [əz] | *roses* |
| past *-ed* | voiceless 뒤 | [t] | *kissed* |
| | voiced 뒤 | [d] | *buzzed* |
| | /t, d/ 뒤 | [ɪd] | *planted* |
| negative *in-* | bilabial 앞 | im- | *impossible* |
| | /l/ 앞 | il- | *illegible* |
| | /r/ 앞 | ir- | *irregular* |
| | elsewhere | in- | *inactive* |

<span style="color:#c53030;">**⚠️ 함정**</span>: allomorph들은 의미가 다른 게 아니라 **같은 의미·기능에 소리만** 다릅니다. *cats* 의 복수형은 [s]이지 [əz]가 아닙니다([əz]는 sibilant 뒤).

### 복습 포인트

- morpheme = 최소 의미·기능 단위. free(혼자 섬) / bound(붙어야 함, 모든 affix 포함).
- inflection = 품사 유지·8개·suffix만. derivation = 새 단어·품사 변경 가능·prefix·suffix.
- 동음 *-er*: comparative(inflection, adj) vs agentive(derivation, V→N).
- word-formation: affix 더함(derivation) / 합침(compounding) / affix 없이 품사 전환(conversion) / 자름(clipping) / 조각 섞음(blending) / 머리글자(acronym·initialism) / affix 제거(backformation).
- compound: head는 rightmost, head의 하위어면 endocentric·아니면 exocentric, 강세·inflection으로 phrase와 구별.
- bracketing은 의미를 가른다 — *un-* 두 morpheme + affix 품사 제약이 순서를 고정.
